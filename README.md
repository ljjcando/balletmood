# Ballet Mood

> 记录每一次优雅的进步

一款专为芭蕾舞者设计的微信小程序，采用莫兰迪燕麦色系，帮助舞者记录每日练习感悟、课程进展，追踪芭蕾术语学习。

---

## 核心功能

### 1. 日历记录
- 月视图日历，直观查看练习记录
- 点击已有记录进入**展示模式**（明信片卡片）
- 点击空白日期进入**编辑模式**（新建记录）
- 有记录的日期显示特殊标记，有图片的日期显示缩略图

### 2. 术语标签
- 内置常用芭蕾术语（Rond de jambe、Adagio、jete 等）
- 支持多选术语标签
- 支持添加自定义术语
- 长按术语可删除

### 3. 多课程展示
- 单日可记录多节课程
- 支持选择课程类型（入门、初级、中级、PBT 等）
- 记录每节课的时长
- 详情页展示所有课程标签，横向排列自动换行

### 4. 图片上传
- 支持从相册或拍照上传练习照片
- 自动裁剪为 3:4 竖向比例
- 详情页有图时显示大图，无图时隐藏图片区域

### 5. 数据统计
- 本月目标设定
- 本地存储，数据永久保存
- 支持编辑和删除历史记录

### 6. 身体部位标记
- 标记练习涉及的身体部位（核心、脚背、外开、背部、手臂）
- 多选胶囊标签形式

## 特别记录：本次优化重点 (v1.1.0)

### 1. 展示模式与编辑模式的分离
**问题**：原逻辑中，点击有图片的记录进入展示模式，无图片记录直接跳转编辑模式。

**优化**：
```javascript
// 统一入口逻辑：只要已有记录，均进入展示模式
if (record) {
  this.openPostcard(date);   // 展示模式
} else {
  this.openEditModal(date);   // 新建模式
}
```

**效果**：用户体验更连贯，可先查看再决定是否编辑。只有点击"重新编辑"按钮才进入编辑态。

---

### 2. 按需渲染的性能优化
**图片区域条件渲染**：
```xml
<view class="postcard-image-wrapper" wx:if="{{postcardData.photo}}">
```
- 无图片记录不渲染图片容器
- 文字内容自动上移，布局紧凑

**课程标签按需展示**：
```xml
<view class="postcard-courses">
  <text class="postcard-course-tag" wx:for="{{postcardData.courses}}" wx:key="index">{{item}}</text>
</view>
```
- 空数组时自动隐藏整个标签区域
- 使用 `filter` 过滤空值，确保只渲染有效数据

- 课程标签移至 "Ballet Mood" 后方，作为品牌后缀
- 形成完整的品牌展示区：`🩰 Ballet Mood [初级] [PBT]`

## 数据结构

### 单条记录格式
```javascript
{
  photo: "cloud://...",         // 图片路径（可选，云端 fileID；旧版本可能残留本地临时路径）
  note: "今天的练习感悟...",     // 文字记录
  terms: ["Adagio", "jete"],   // 术语数组
  bodyParts: ["核心 Core"],     // 身体部位数组
  courses: [                    // 课程数组
    { name: "初级", duration: "1.5" },
    { name: "PBT", duration: "1.0" }
  ],
  updatedAt: "2025-03-21T..."
}
```

### 图片兼容说明
- 1.0.3 及后续版本会优先将图片上传到云端，并在记录中保存 `cloud://...` 文件 ID。
- 旧版本 1.0.2 及更早版本，部分记录里可能仍是手机本地临时路径。
- 当这些旧路径失效时，记录本身仍会保留，界面会提示：`图片已过期，请重新上传`。

## 使用说明

1. **新建记录**：点击日历上的空白日期
2. **查看记录**：点击有记录的日期，进入展示模式
3. **编辑记录**：在展示卡片中点击"重新编辑"
4. **删除记录**：展示模式或编辑模式中均可删除
5. **切换月份**：点击顶部左右箭头

## 云开发索引配置

当前版本已经使用结构化云同步，涉及 3 个集合：

- `ballet_mood_records`
- `ballet_mood_profiles`
- `ballet_mood_users`（旧版兼容用 legacy 集合）

这些索引不是仓库里的代码文件，必须在微信开发者工具或云开发控制台里手动创建。

### 推荐索引

#### 1. `ballet_mood_records`

用途：按当前用户读取全部日记记录，是最重要的一组索引。

建议创建：

- 组合索引：`_openid` 升序，`dateKey` 升序
- 组合索引：`_openid` 升序，`updatedAt` 降序

说明：

- `_openid + dateKey` 适合按用户维度维护每天唯一记录，也方便后续按日期做精确查找。
- `_openid + updatedAt` 适合按用户读取最近更新的记录，后续如果做“最近编辑”“增量同步”会更稳。

#### 2. `ballet_mood_profiles`

用途：按当前用户读取术语、目标、课程标签等个人配置。

建议创建：

- 组合索引：`_openid` 升序，`updatedAt` 降序

说明：

- 当前代码会按 `_openid` 读取当前用户 profile，并在多条记录并存时优先挑最新一条。

#### 3. `ballet_mood_users`

用途：兼容旧版本单文档快照恢复。

建议创建：

- 组合索引：`_openid` 升序，`updatedAt` 降序
- 单字段索引：`updatedAt` 降序

说明：

- `_openid + updatedAt` 用于优先查当前用户的最新 legacy 快照。
- 单独的 `updatedAt` 索引用于“同用户查不到时，退回全局最新文档”的兼容逻辑。

### 创建步骤

1. 打开微信开发者工具。
2. 进入“云开发”面板。
3. 打开对应环境。
4. 进入“数据库”。
5. 选择集合，例如 `ballet_mood_records`。
6. 打开“索引”页签。
7. 按上面的字段顺序创建索引。
8. 等索引状态变成“可用”后，再创建下一条。

### 最小必建集合

如果你这次只想先补最关键的，优先级是：

1. `ballet_mood_records`: `_openid` + `dateKey`
2. `ballet_mood_profiles`: `_openid` + `updatedAt`
3. `ballet_mood_users`: `_openid` + `updatedAt`

先把这 3 条建上，当前版本的启动恢复、删除同步和刷新流程就够用了。

## 项目更新记录

### v1.0.3 发布记录 (2026-04-11)

#### 本次更新内容
- 云同步从单文档快照补强为结构化同步，拆分为 `ballet_mood_profiles` 与 `ballet_mood_records`
- 启动时优先展示本地数据，再在后台继续做云端迁移或恢复，减少首页打开等待
- 新保存的图片会优先转成可持久化路径，并在同步时上传为 `cloud://...` 文件 ID
- 旧版本遗留的失效图片不再直接显示坏图，改为提示“图片已过期，请重新上传”
- 删除记录时会同步更新页面、本地存储与云端记录，避免删除后刷新又恢复
- 云开发数据库已补充本轮需要的关键索引，降低按用户查记录与恢复历史时的扫描成本

#### 本次解决的 Bug / 逻辑问题
- 修复微信小程序缓存清理后，历史记录无法稳定从云端恢复的问题
- 修复同一天存在多条云记录时，可能拿到较差版本记录的问题
- 修复云端较新的记录照片为空时，把本地有效照片覆盖掉的问题
- 修复保存后云端返回新的 `cloud://...` 图片 ID，但页面仍保留旧本地路径的问题
- 修复删除记录后界面提示成功，但日历缩略图和刷新结果仍残留的问题
- 修复课程标签可能重复保存、重复展示的问题

#### 修复总结
- 这一轮重点不在新功能，而在“记录链路闭环”。
- 现在一条记录从新建、编辑、删除，到本地保存、云端同步、启动恢复，链路已经统一。
- 图片处理也从“依赖本地临时路径”改成了“优先持久化 + 云端 fileID”，降低了后续丢图概率。
- 对旧数据保留兼容逻辑，已经存过的历史记录不会因为版本升级直接失效。

#### 发版备注
- 建议升级说明：本次版本重点修复历史记录恢复、图片过期提示与删除同步问题。
